Laboratorio 2.1: Actualizar el sistema
==================================================================================================================================

1- Actualize los repositorios del sistema (update & Upgrade) y Enumere los repositorios actualmente en uso en su servidor.
2- Proceda a buscar la herramienta en el cache de los repositorios a ver si se encuentra disponible.
3- Instale la herramienta "Bashtop Resource Monitor" en caso de ser necesario, agregarla al repositorio local (https://es.linux-console.net/?p=438#gsc.tab=0)
4- Una vez halla presentado la herramienta anterior, proceda a desinstalarla. eliminando tambien todos los archivos de configuracion 
5- Elimine todas las dependencias asociadas a este programa que ya no esten en uso

## Bashtop es una herramienta de monitoreo de recursos del sistema que se ejecuta en la terminal y permite visualizar en tiempo real el estado del servidor o computadora. Sirve para supervisar el uso de CPU, memoria RAM, procesos activos, red y almacenamiento, todo desde una interfaz interactiva y gráfica dentro de la consola. Es útil para administradores y técnicos porque facilita detectar procesos que consumen demasiados recursos, analizar rendimiento del sistema y diagnosticar problemas sin necesidad de entorno gráfico.

# Mostrar la lista de repositorios actualmente habilitados en el servidor.
# Permite verificar qué fuentes de software están activas antes de actualizar.
dnf repolist

# Actualizar metadatos e instalar versiones más recientes de paquetes ya instalados.
# Se usa para sincronizar el sistema con los repositorios.
dnf update

# Realizar una actualización completa del sistema.
# Similar a update, pero fuerza actualización más agresiva de paquetes/obsoletos.
dnf upgrade

# Volver a listar repositorios después de actualizar para confirmar disponibilidad.
dnf repolist

# Buscar en el caché de repositorios si el paquete "bashtop" está disponible.
# Esto permite validar si puede instalarse directamente desde repositorios.
dnf search bashtop

# Instalar Git si no estaba previamente disponible.
# Git será necesario para clonar el repositorio del proyecto desde GitHub.
dnf install git -y

# Clonar el repositorio oficial de Bashtop desde GitHub.
# Se descarga el código fuente porque el paquete no estaba disponible en repositorios.
git clone https://github.com/aristocratos/bashtop.git

# Listar archivos del directorio actual para confirmar que el repositorio fue descargado.
ls

# Entrar en la carpeta del proyecto Bashtop descargado.
cd bashtop

# Ejecutar instalación de Bashtop desde el código fuente.
# Este comando copia binarios, archivos de configuración y recursos al sistema.
make install

# Ejecutar Bashtop para verificar que fue instalado correctamente.
# Se abre el monitor de recursos interactivo en terminal.
bashtop

# Ejecutar Bashtop directamente desde el directorio actual.
# Otra forma de lanzar el programa si el binario está en la carpeta local.
./bashtop

# Intentar desinstalar Bashtop usando DNF.
# Este comando solo funcionaría si el programa hubiera sido instalado como paquete RPM.
dnf remove bashtop

# Eliminar dependencias huérfanas que ya no son necesarias.
# Limpia paquetes instalados automáticamente que quedaron sin uso.
dnf autoremove

# Buscar archivos de configuración de Bashtop en /etc.
# Permite verificar si quedaron configuraciones residuales tras la desinstalación.
ls /etc/bashtop*

# Buscar manualmente archivos relacionados con Bashtop en /etc usando grep.
# Método alternativo para confirmar residuos de configuración.
ls /etc | grep -i bashtop

# Mostrar detalles completos de archivos de configuración encontrados.
ls -l /etc/bashtop*

# Verificar si Bashtop sigue apareciendo como paquete instalado.
# Si no devuelve resultados, ya no está registrado como instalado en DNF.
dnf list installed | grep -i bashtop

# Buscar la ruta del ejecutable de Bashtop.
# Si no devuelve ruta, significa que el binario ya no está disponible en PATH.
which bashtop

Laboratorio 2.2: Programar tareas
=======================================================================================================================

Cron sirve para programar tareas que se ejecutan de forma repetitiva en horarios definidos, mientras que at permite programar una tarea para ejecutarse una sola vez en un momento específico. En resumen: Cron = tareas recurrentes / At = tarea única.

1-Cree un trabajo cron que actualice el sistema todas las tardes a las 11 p.m
2- Programe su máquina para que se reinicie todos los domingos a las 3 a.m.
3- Cree una tarea programada usando el comando "at" para que elimine el contenido de la carpeta /tmp dentro de 1 minuto. Mostrar el resultado con ls /tmp antes y despues de la ejecucion 

# Obtener una shell de superusuario.
# Esto permite trabajar con privilegios administrativos para configurar tareas programadas del sistema.
sudo su

# Abrir el editor del crontab del usuario/root.
# Aquí se agregan tareas programadas recurrentes que se ejecutarán automáticamente.
crontab -e

# ┌──────── Minuto (0 - 59)
# │ ┌────── Hora (0 - 23)
# │ │ ┌──── Día del mes (1 - 31)
# │ │ │ ┌── Mes (1 - 12)
# │ │ │ │ ┌ Día de la semana (0 - 7) → 0 y 7 = Domingo
# │ │ │ │ │
# * * * * * comando

# Dentro del archivo crontab se agregan las siguientes tareas:

# Actualizar el sistema todos los días a las 11:00 PM
0 23 * * * dnf update -y && dnf upgrade -y

# Reiniciar el sistema todos los domingos a las 3:00 AM
0 3 * * 0 reboot

# Iniciar el servicio atd.
# Este servicio permite ejecutar tareas únicas programadas con el comando at.
systemctl start atd

# Habilitar atd para que inicie automáticamente con el sistema.
systemctl enable atd

# Mostrar contenido actual de /tmp antes de la ejecución programada.
# Esto permite evidenciar qué archivos existen antes de la limpieza.
ls /tmp

# Programar una tarea única para ejecutarse en 1 minuto.
# El comando eliminará todo el contenido de la carpeta /tmp.
echo "rm -rf /tmp/*" | at now +1 minute

# Mostrar nuevamente el contenido de /tmp después de esperar la ejecución.
# Sirve para verificar que la tarea programada eliminó correctamente los archivos.
ls /tmp

Comandos Práctica 3: Gestión de Discos, Particiones y Montaje de Sistemas de Archivos
===================================================================================================================================

# Mostrar todos los discos y particiones detectados por el sistema.
# Esto permite identificar el nuevo disco agregado a la máquina virtual.
lsblk

# Iniciar el particionado del nuevo disco.
# Después de /dev/ se coloca el nombre del disco agregado, en este caso nvme0n2.
fdisk /dev/nvme0n2

# Dentro de fdisk se siguió este proceso para crear la partición:
# n  -> Crear nueva partición
# p  -> Definirla como partición primaria
# 1  -> Número de partición (partición 1)
# Enter -> Aceptar primer sector por defecto
# Enter -> Aceptar último sector por defecto (usar todo el espacio disponible)
# w  -> Guardar cambios y salir de fdisk

# Verificar que la nueva partición fue creada correctamente.
# Debe aparecer como /dev/nvme0n2p1.
lsblk

# Intento inicial de formatear la partición como XFS.
# Este formato no correspondía al solicitado en la práctica.
mkfs.xfs /dev/nvme0n2p1

# Formatear correctamente la partición con sistema de archivos EXT4.
# Esto prepara la partición para almacenar datos.
mkfs.ext4 /dev/nvme0n2p1

# Mostrar el directorio actual de trabajo.
pwd

# Listar contenido del directorio actual.
ls

# Entrar al escritorio del usuario.
cd Escritorio

# Crear carpeta que servirá como punto de montaje inicial.
mkdir datos

# Montar la partición en la carpeta datos.
mount /dev/nvme0n2p1 /home/egalan/Escritorio/datos

# Verificar contenido del escritorio.
ls

# Entrar al disco montado.
cd datos

# Crear archivo de prueba dentro de la partición.
touch AdrianAlcantara.txt

# Verificar creación del archivo.
ls

# Regresar al directorio anterior.
cd ..

# Intento incorrecto de desmontar.
unmount /dev/nvme0n2p1

# Desmontar correctamente la partición.
umount /dev/nvme0n2p1

# Montar la misma partición en /mnt.
mount /dev/nvme0n2p1 /mnt

# Entrar al nuevo punto de montaje.
cd /mnt

# Verificar que el archivo sigue existiendo.
ls

# Desmontar nuevamente la unidad.
umount /mnt
